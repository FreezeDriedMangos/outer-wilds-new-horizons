Body Schema

Type: object

A planet or body to generate

Root Name
Type: string
Default: "New planet"

Unique name of your planet

Root StarSystem
Type: string
Default: "SolarSystem"

Unique star system containing your planet

Root Destroy
Type: boolean
Default: false

True if you want to delete this planet

Root Base
Type: object

Root Base HasMapMarker
Type: boolean
Default: false

If the body should have a marker on the map screen.

Root Base HasAmbientLight
Type: boolean
Default: false

If the dark side of the body should have some slight ammount of light

Root Base SurfaceGravity
Type: number
Default: 12

The acceleration due to gravity felt as the surfaceSize. Timber Hearth has 12 for reference

Root Base GravityFallOff
Type: enum (of string)
Default: "linear"

How gravity falls off with distance. Most planets use linear but the sun and some moons use inverseSquared.

Must be one of:

  • "linear"
  • "inverseSquared"

Root Base SurfaceSize
Type: number
Default: 100

A scale height used for a number of things. Should be the approximate radius of the body.

Root Base SphereOfInfluence
Type: number
Default: 0

An override for the radius of the planet's gravitational sphere of influence. Optional

Root Base GroundSize
Type: number
Default: 0

Radius of the a simple sphere used as the ground for the planet. If you want to use more complex terrain, leave this as 0.

Root Base HasCometTail
Type: boolean
Default: false

If you want the body to have a tail like the Interloper.

Root Base HasReferenceFrame
Type: boolean
Default: true

Allows the object to be targeted on the map.

Root Base CenterOfSolarSystem
Type: boolean
Default: false

Set this to true if you are replacing the sun with a different body. Only one object in a star system should ever have this set to true.

Root Base IsSatellite
Type: boolean
Default: false

Is this body an artificial satellite of a planet/moon/star?

Root Orbit
Type: object

Root Orbit SemiMajorAxis
Type: integer
Default: 5000

The semi-major axis of the ellipse that is the body's orbit. For a circular orbit this is the radius.

Value must be greater or equal to 0

Root Orbit Inclination
Type: number
Default: 0

The angle (in degrees) between the body's orbit and the plane of the star system

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbit PrimaryBody
Type: string
Default: "Sun"

The name of the body this one will orbit around

Root Orbit IsMoon
Type: boolean

Is this the moon of a planet? Used for determining when its name is shown on the map.

Root Orbit LongitudeOfAscendingNode
Type: number
Default: 0

An angle (in degrees) defining the point where the orbit of the body rises above the orbital plane if it has nonzero inclination.

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbit Eccentricity
Type: number
Default: 0

At 0 the orbit is a circle. The closer to 1 it is, the more oval-shaped the orbit is.

Value must be greater or equal to 0 and lesser or equal to 0.99999

Root Orbit ArgumentOfPeriapsis
Type: number
Default: 0

An angle (in degrees) defining the location of the periapsis (closest distance to it's primary body) if it has nonzero eccentricity.

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbit TrueAnomaly
Type: number
Default: 0

Where the planet should start off in its orbit in terms of the central angle.

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbit AxialTilt
Type: number
Default: 0

The angle between the normal to the orbital plane and its axis of rotation.

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbit SiderealPeriod
Type: number
Default: 0

Rotation period in minutes.

Root Orbit IsTidallyLocked
Type: boolean
Default: false

Should the body always have one side facing its primary?

Root Orbit ShowOrbitLine
Type: boolean
Default: false

Referring to the orbit line in the map screen.

Root Orbit IsStatic
Type: boolean
Default: false

Is the body meant to stay in one place without moving?

Root Orbit Tint
Type: object

Colour of the orbit-line in the map view.

Root Orbit Tint R
Type: integer
Default: 0

Value must be greater or equal to 0

Root Orbit Tint G
Type: integer
Default: 0

Value must be greater or equal to 0

Root Orbit Tint B
Type: integer
Default: 0

Value must be greater or equal to 0

Root Orbit Tint A
Type: integer
Default: 0

Value must be greater or equal to 0

Root Orbit TrackingOrbitLine
Type: boolean
Default: false

Should we just draw a line behind its orbit instead of the entire circle/ellipse?
